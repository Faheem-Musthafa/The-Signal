
# The Signal

**The Signal** is an AI-powered executive briefing generator. It lets users select topics, then synthesizes real-time intelligence into concise, shareable digests using LLMs and live web crawling. Built with Next.js, Convex, Clerk, and Firecrawl, it features robust caching, per-user quotas, and a modern, minimal UI.

## Features

- 🔒 Clerk authentication and per-user daily quota (3 digests/day)
- ⚡️ Real-time topic-based digest generation (Firecrawl + OpenRouter LLM)
- 🗂️ Convex-powered persistence, caching, and history
- 🔁 3-hour cache reuse to save tokens/credits
- 📋 “Previously Crawled Today” history with deduplication
- 🚦 Rate limiting (per-account and IP)
- 🔗 Shareable digests with unique links
- 🛡️ Production-ready: typechecked, linted, and hardened for deployment


## What is implemented

- Next.js 14 app with TypeScript and Tailwind baseline
- `POST /api/digest` with:
  - topic allowlist validation (1-8 topics)
  - Clerk-authenticated access only
  - per-account daily cap (3 digests/day)
  - Convex-based rate limiting call (best-effort for MVP)
  - Firecrawl + OpenRouter primary path
  - OpenRouter model fallback rotation (if a model endpoint is unavailable)
  - retry once on upstream failure (5s delay)
  - robust JSON extraction + Zod story validation
  - Convex digest persistence (`saveDigest`)
- `GET /api/health/llm` to validate LLM env, key-shape warnings, and model availability
- `POST /api/share` for explicit snapshot creation
- Convex schema tables: `digests`, `rateLimits`, `subscribers`, `emailQueue`
- Convex functions:
  - `digests.saveDigest`, `digests.getByShareId`
  - `rateLimits.checkAndIncrement`
  - `internal.cleanup.purgeExpiredDigests`
  - `crons.ts` scheduled jobs wiring
  - Phase 3 placeholder `internal.subscribers.sendDueDigests`

## Setup

1. Install dependencies:

```bash
npm install
```

2. Add env vars from `.env.example` into `.env.local`.

Required Clerk env vars:

```bash
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=
CLERK_SECRET_KEY=
```

3. Initialize Convex and generate server/client bindings:

```bash
npx convex dev
```

4. Run the app:

```bash
npm run dev
```

## Notes

- UI/UX is intentionally minimal so you can design it yourself.
- Digest generation requires `OPENROUTER_API_KEY` and `FIRECRAWL_API_KEY`.
- Optional env vars:
  - `OPENROUTER_MODEL_FALLBACKS` (comma-separated model IDs)
  - `OPENROUTER_MAX_TOKENS` (default `2000`, capped in code)
  - `FIRECRAWL_COUNTRY` (default `US`)
  - `RATE_LIMIT_MAX` and `RATE_LIMIT_WINDOW` (IP-based best-effort limiter)
- `api` imports under `@convex/_generated/*` are generated by Convex; run `npx convex dev` first.
- If Convex write fails, `/api/digest` still returns stories without `shareId`.
